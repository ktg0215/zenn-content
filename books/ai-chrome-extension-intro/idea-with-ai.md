---

## 「何を作るか」はいつも悩む

個人開発では何でも簡単に作れるようになったからこそ、何を作るかで悩みます。日々、布団の中で「何を作るべきか」を考えてしまう。そんな経験がある方も多いのではないでしょうか。

ただ、悩み続けていると「いずれ悩むのもやめて、何も開発しなくなる」という状態に陥ります。これは私の経験談です。だから、考え方のフレームを持つことが重要です。

## 最強のアイデア源：自分のストレス

一番手っ取り早いのは、**自分がストレスに感じていることや「あったらいいな」を作ること**です。

自分の理想を追えるので、ゴールがわかりやすい。「使い続けたいか」の検証コストがゼロ。私が作った拡張の大半は、自分の不満から生まれています。

**実例：**

- **YouTubeショートが邪魔だったから消した** → YouTube Shorts Killer（現在インストール数11,000+）
- **引っ越しのタイミングで物件探しを効率化したかった** → 物件分析（賃貸・購入2本）
- **Pocketが重くてシンプルなブックマークが欲しかった** → ReadMark（現在6,200+）

物件分析は、引っ越しのタイミングじゃなければ思いつかなかったと思います。自分が当事者だからこそ生まれたアイデアです。

### 「ストレス棚卸し」の方法

今日1日を振り返って、「これがもう少し楽になれば」と感じた瞬間を書き出してみてください。

- Webブラウジング中の不満
- 繰り返している手作業
- 特定のサイトで「これができたら」と思ったこと

この「不満リスト」が、あなたのアイデアプールです。

## AIを使ったアイデア発掘プロンプト

自分のストレスをAIに話しかけるだけで、具体的な機能提案が返ってきます。

```
私は毎日〇〇という作業に時間がかかっています。
Chromeブラウザ上でこれを効率化するアイデアを10個提案してください。
既存のChrome拡張機能で解決できないものを優先してください。
```

このプロンプトで出てきたアイデアを、以下の基準で絞り込みます。

1. **Chrome Web Storeで検索して競合を確認** — すでに強力な競合がいるかどうか
2. **自分が使いたいか** — ターゲットユーザーが自分なら検証コストゼロ
3. **AIに「これはChrome拡張で実装可能か？」と確認** — 技術的な実現可能性

### AIに聞くときの注意点

「同じことを聞いた人は、同じ答えにたどり着きやすい」という問題があります。Claude、ChatGPT、Geminiなど複数のAIで試してみると、それぞれ違う答えを返してくることが多いです。

また、**興味がないジャンルのアイデアは避ける**ことも重要です。自分の「こうしたい」がないと、完全にAIの言いなりになりがちで、既存アプリと似たようなものになってしまいます。

## 個人開発で最も大事なのは「技術」でも「知識」でもなく「情報」

ここで私が確信していることをお伝えします。個人開発で今後必要になるのは、技術でも知識でもなく**「情報」**です。

| 種類 | 内容 |
|-----|-----|
| 知識 | 仕組みを理解している（How it works） |
| 情報 | できることを知っている（What's possible） |

例えば、Chrome拡張のアップデートはAPIを使ってAIが自動で行えます。私は仕組みを詳しく知りません。AIに「自動でアップロードして」と言ったらやってくれる。

知っている人からしたら当たり前のことですが、知らない人も多い。**「何ができるか」を知っていることが、個人開発では最大の武器**になります。

### 情報の仕入れ方

具体的な情報収集の場：

- **Chrome Web Storeのレビュー** — 競合拡張への不満が宝の山
- **Reddit（r/chrome_extensions）** — 英語圏ユーザーの課題感が分かる
- **Twitter/X・Zenn・Qiita** — 日本語圏の「困った」を探す
- **AIに「これ自動化できない？」と聞く** — できることの発見

私がCookieJarを開発したのも、かつて人気だったEditThisCookieという拡張機能が2024年に更新停止になり、代替を求めるユーザーが多数いると知ったからです。「情報」が先にあって、プロダクトが後に来ました。

## アイデアを検証するたった2つのチェック

アイデアが浮かんだら、作り始める前にこの2つだけ確認します。

**チェック1: Chrome Web Storeで検索**

同じキーワードで検索して出てくる競合を分析します。上位5本のインストール数と最終更新日を確認するだけで、市場の状況が見えてきます。

- インストール数10万以上の競合が複数 → 差別化が難しい
- 最終更新が1年以上前 → 放置されている＝チャンス

**チェック2: AIに「このアイデアの問題点は？」と聞く**

盲点を先に潰します。「技術的に不可能な理由」「既存の代替手段」「ユーザーが使わない理由」——こういった観点でAIに批判的に評価してもらうと、致命的な問題を事前に発見できます。

## まとめ：アイデアの発掘から検証まで

```
1. 自分のストレスを書き出す（ストレス棚卸し）
2. AIに「〇〇をChrome拡張で解決するアイデアを10個」と聞く
3. CWSで競合を検索する
4. AIに「問題点は？」と聞いて盲点を潰す
5. 残ったものを作る
```

このフローを1〜2時間でこなせるようになれば、「何を作るか」で悩む時間が大幅に減ります。
