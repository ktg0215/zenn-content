### 案A：「ターミナル＋拡張」ミニマル型

**テーマ**: コードとブラウザの融合。「作る」行為そのものを表現。

```
背景: #0F1117（ほぼ黒）
アクセント: #3B82F6（青）
レイアウト:
  - 上部1/3: ターミナル画面のスクリーンショット風（暗背景にコード行）
  - 下部2/3: タイトルテキスト（白）+ サブタイトル（グレー）
  - 右下隅: Chromeパズルアイコン（青）
フォント: Geist Mono（タイトル大）/ Noto Sans JP（サブタイトル）
```

**キャッチ**: `npm run build` の下に「あなたのアイデアが拡張機能になる」

---

### 案B：「17本の拡張アイコン」グリッド型

**テーマ**: 実績を視覚化。17本のアイコンが並ぶことで「量産できる」が伝わる。

```
背景: #F8F8F8（オフホワイト）
アクセント: #1A1A1A（ほぼ黒）
レイアウト:
  - 上部: 4×4グリッドに16個の拡張アイコン（小さく）
  - 中央: 縦線で区切り
  - 下部: タイトルテキスト（黒、大）+ 著者名
フォント: Geist Sans（タイトル）/ Noto Sans JP（説明）
```

**キャッチ**: 「コードが書けなくても」を斜線でゆるく装飾

---

### 案C：「AIとブラウザ」対話型（推奨）

**テーマ**: AIとの対話から拡張が生まれるプロセスを象徴。最もターゲット読者に刺さる。

```
背景: グラデーション（#0A0A0A → #1A2030、上から下）
アクセント: #A3E635（ライムグリーン）
レイアウト:
  - 上部: チャット吹き出し2つ（AIとユーザーの会話を示唆）
    - 左: 「Chrome拡張を作りたい」（白テキスト、グレー背景）
    - 右: 「manifest.json を...」（白テキスト、緑背景）
  - 中央: 矢印（↓）+ 「1週間で公開」
  - 下部: タイトルテキスト（白、大）
フォント: Geist Sans（タイトル）/ Noto Sans JP（サブ）
```

**キャッチ**: 「AIと対話するだけ」を视覚化
