## 公開直後に感じる「無」

公開した直後、ほとんどの場合インストール数は0のままです。CWSは公開しただけでは誰にも発見されません。これは仕様です。

ここで「需要がなかった」と諦めるのは早計です。まだ誰にも知られていないだけです。

マーケティング戦略を考えたところで、その戦略が通用するステージまで辿り着くのが難しい。「分析して改善しよう」の前提条件は、分析できる実績があることです。まずは最初の30インストールを獲得することに集中します。

## 最初の30インストールを獲得する

### 効果があった方法

**① Zenn・Qiitaに開発記事を書く**

「なぜ作ったか」「どんな技術を使ったか」を書くだけで流入します。記事内にCWSリンクを貼ると、記事を読んだ人がそのままインストールしてくれます。

**② X（Twitter）で告知する**

`#個人開発` `#Chrome拡張` ハッシュタグを使って投稿します。100回発信して1回バズればいい——でも発信しなければその1回は絶対に来ません。

**③ ProductHuntに投稿する**

英語圏向けのプロダクト発見プラットフォームです。うまくいけば数日で数百インストールを獲得できます。

**④ 関連するRedditに投稿する**

`r/chrome_extensions`、`r/productivity`、`r/nosurf` などのコミュニティで紹介します。英語での投稿が必要ですが、AIに翻訳を依頼すれば問題ありません。

### 効果がなかった方法

- **公開して待つだけ** — 何も起きません
- **有料広告** — 個人開発の拡張機能には費用対効果が合わない

## CWS SEOで継続的な流入を作る

最も費用対効果が高い施策は**タイトルのキーワード最適化**です。

### 実例：Procshotのタイトル変更

**変更前**：`Procshot`

**変更後**：`Procshot — Automatic Procedure Capture & Step-by-Step Guide Maker`

この変更だけで、検索インプレッションが**340%増加**しました。タイトルの変更は5分でできます。今すぐやる価値があります。

### タイトルの最適化ルール

```
[ブランド名] — [機能キーワード] [ユースケース]

Good: "ReadMark — Reading Position Saver for Long Articles"
Bad:  "ReadMark"
```

Short descriptionも同様に、自然な文章の中にキーワードを含めます。

### ローカライゼーションでインデックスを増やす

Chrome Web Storeは言語ごとに別々の検索インデックスを持ちます。8言語に対応すると、事実上8倍の検索機会が生まれます。

```
18本の拡張 × 8言語 = 136の検索可能なリスティング
```

AIに翻訳を依頼すれば、1本の拡張を8言語に対応するのに1〜2時間で完了します。

## 「量より質」より「量で質を学ぶ」

18本作って確信したのは、**1本をピカピカに磨くより、10本を作る方が学びが大きい**ということです。

- 2本目は1本目より50%速く作れる
- 5本目になると「ユーザーが何を求めているか」の勘が養われる
- 18本になると「当たり筋」と「外れ筋」のパターンが見えてくる

最初の1本に時間をかけすぎないことが、長期的に成功する秘訣です。「ひとつのアプリをピカピカに磨くより、多数のアプリをリリースした方が何かしら当たる可能性が高い」——これは量産フローを続けた末に至った結論です。

結果が出るタイミングはわかりません。リリースしてすぐに伸びることもあれば、半年後に急に伸びることもある。何年経っても伸びないこともある。だから「伸びないから失敗」とも言い切れない。

**唯一わかったこと：発信し続けないと、何も起きない。**

## GA4で数字を見る

どの拡張に注力すべきかは感覚ではなく数字で判断します。

全拡張にGA4のMeasurement Protocolを実装して、以下を計測します：

```typescript
// src/background.ts
const GA4_ENDPOINT = 'https://www.google-analytics.com/mp/collect';
const MEASUREMENT_ID = 'G-XXXXXXXXXX';
const API_SECRET = 'YOUR_SECRET';

async function sendGA4Event(eventName: string, params: Record<string, string | number>) {
  await fetch(`${GA4_ENDPOINT}?measurement_id=${MEASUREMENT_ID}&api_secret=${API_SECRET}`, {
    method: 'POST',
    body: JSON.stringify({
      client_id: await getAnonymousClientId(),
      events: [{ name: eventName, params }],
    }),
  });
}

// 計測すべきイベント
// install, update: インストール・更新時
// feature_used: 主要機能の使用（どの機能が使われているか）
// paywall_shown: ペイウォール表示（転換率の分母）
// review_prompt_shown/clicked/dismissed: レビュー依頼への反応
```

数字を見ることで「どの拡張が本当に使われているか」「どの機能が価値を提供しているか」が明確になります。
