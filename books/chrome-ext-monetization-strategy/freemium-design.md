## フリーミアムの3つの型

Chrome拡張のフリーミアムには大きく3つの型があります。

| 型 | 仕組み | 向いているケース |
|---|-------|--------------|
| **機能制限型** | Free版では一部機能のみ使える | 機能群が明確に分離できる拡張 |
| **使用回数制限型** | 月N回まで無料、以降はPro | 操作1回ごとに価値が発生する拡張 |
| **リバーストライアル型** | 最初N日間は全機能→以降Free制限 | 新規ユーザーに価値を体験させたい |

### 機能制限型

最もシンプルで実装しやすい。Free版で核となる機能を提供し、「より便利な機能」をPro版に置く。

**メリット**: Free版ユーザーが長期にわたって使い続けやすい（離脱が少ない）
**デメリット**: Pro機能の価値がFree版のまま伝わらない（使ってみなければわからない）

### 使用回数制限型

「今月あと3回しか使えません」という表示が転換の動線になる。

**メリット**: 価値を感じるタイミング（上限に達したとき）でアップグレードを促せる
**デメリット**: 回数リセット（月初め）を待つユーザーが多く、転換せずに継続使用される場合がある

### リバーストライアル型（S-Hub採用）

インストール後14日間はPro版と全く同じ機能を提供し、14日後にFree版に自動ダウングレードします。

**メリット**: ユーザーがProの価値を体験してから判断できる。「失うことへの恐れ（Loss Aversion）」が転換動機になる
**デメリット**: 実装コストが最も高い。「14日後に払うかどうか」を判断するユーザーが、14日間の価値を忘れていることがある

## リバーストライアルの仕組みと効果

S-HubではProcshot・FocusGuard・DataPickでリバーストライアルを採用しています。

### Procshotでの実験結果

2025年末に、フリーミアムモデルをリバーストライアル型に切り替えた結果です。

| 指標 | 変更前（機能制限型） | 変更後（リバーストライアル型） |
|-----|----------------|---------------------|
| 有料転換率 | 1.9% | 3.2% |
| 平均課金までの日数 | 23日 | 11日 |
| 14日以内の転換比率 | 31% | 67% |
| 返金率 | 2.1% | 3.8% |

転換率が**68%向上**しました。ただし返金率も上昇しています。「体験して気に入ったから払う」より「失いたくないから払う」が転換動機になるため、期待値とのギャップが生じやすい面があります。

### 実装のポイント

リバーストライアルを実装する場合、インストール日時をchrome.storage.localに保存し、14日後にFreeモードへ切り替えます。

```typescript
// background.ts — リバーストライアルの状態管理
const TRIAL_DAYS = 14;

async function checkTrialStatus(): Promise<'trial' | 'pro' | 'free'> {
  const { installDate, isPro } = await chrome.storage.local.get([
    'installDate',
    'isPro',
  ]);

  // ExtensionPayで有料確認済みの場合
  if (isPro) return 'pro';

  // 初回インストール時にinstallDateを記録
  if (!installDate) {
    await chrome.storage.local.set({ installDate: Date.now() });
    return 'trial';
  }

  const daysSinceInstall = (Date.now() - installDate) / (1000 * 60 * 60 * 24);
  return daysSinceInstall <= TRIAL_DAYS ? 'trial' : 'free';
}

// 機能呼び出し時のガード
export async function requireProOrTrial(): Promise<boolean> {
  const status = await checkTrialStatus();
  return status === 'pro' || status === 'trial';
}
```

## 境界設計の5原則

どの型を選ぶにしても、境界線の引き方に以下の5原則が適用されます。

### P1: Free版だけでも十分価値がある

Free版で「使えるが不十分」ではなく「使えて満足」を提供します。ユーザーを騙した感覚にさせると、レビューで低評価をつけられます。

**NG**: Free版でダウンロードボタンを表示するが押すとペイウォール（機能をちらつかせて使わせない）
**OK**: Free版でCSV5件分はエクスポートできる（価値を体験させた上で制限）

### P2: Pro版のメリットが一目瞭然

Free版を使っているときに「Proならこれができる」が自然に見える。機能比較表・グレーアウトされたPro機能・ツールチップ表示が有効です。

### P3: アップグレード動線が自然

「もう少し使いたい」と思ったタイミングで、アップグレードボタンが目に入る。上限に達した瞬間・エクスポートしようとした瞬間・高度な設定を開いた瞬間が最も転換率が高い。

### P4: Free→Proの心理的ハードルは低く

価格を最初に見せすぎない。機能を体験→上限に達する→そのときに価格提示、の順が理想。「月$3.99」ではなく「1日13円」に換算する表現も有効です。

### P5: Free→Free継続でも離脱させない

転換しなかったユーザーを「失敗」として扱わない。Free版で長く使い続けるユーザーは口コミとレビューの源泉です。ペイウォールを連発すると離脱が増え、レビューが下がります。

## 境界の具体例：S-Hub 5拡張の実例

| 拡張 | Free版 | Pro版（$3.99/月） | 転換率 |
|-----|-------|----------------|-------|
| **DataPick** | テーブル選択・プレビューのみ | CSV/Excel/Google Sheets エクスポート無制限 | 3.8% |
| **FocusGuard** | ブロックリスト5件 | ブロックリスト無制限 + スケジュール機能 | 2.7% |
| **Procshot** | ガイド5件まで保存 | 無制限保存 + カスタムテンプレート + PDFエクスポート | 3.2% |
| **ReadMark** | スクロール位置保存 | ハイライト機能 + マルチデバイス同期 | 1.4% |
| **JFF** | 全機能無料 | — | 0%（意図的） |

**ReadMarkの転換率が低い理由**: スクロール位置保存という「核となる機能」が無料で提供されており、ハイライトは「あれば便利」にとどまっている。Pro機能の価値が弱い設計ミス。現在リデザイン検討中。

**JFFが全機能無料の理由**: 日本語フォント検出という「ニッチかつ一発系」の機能で、繰り返し使用が少ない。有料化しても転換率が低いと判断し、インストール数最大化（SEO効果・クロスプロモーション流入源）として運用している。

## ペイウォール表示のタイミング

いつペイウォールを見せるかは、転換率に直結します。

### 見せるべきタイミング

- **使用上限に達した瞬間**: 「今日の利用上限に達しました」直後のペイウォールは転換率が最高（FocusGuardで計測：他のタイミングの3.2倍）
- **高付加価値操作の直前**: エクスポートボタンを押したとき、テンプレートを選択したとき
- **価値を体験してから2〜3日後**: インストール直後ではなく、実際に使ってもらってから

### 見せてはいけないタイミング

- **インストール直後の初回起動**: まだ価値を体験していない。疎外感を生みレビューが下がる
- **毎回の起動時**: ウザいと感じたユーザーが削除する
- **エラー発生直後**: 「バグの次にお金を要求される」という最悪の体験

## ペイウォールUIのベストプラクティス

### ソフトブロック vs ハードブロック

**ソフトブロック**: 機能は一部使えるが、制限がかかる。クリックするとペイウォールが表示される。
**ハードブロック**: 操作自体を物理的にブロック。閉じられない、使えない状態にする。

**推奨**: ソフトブロック。ハードブロックはユーザーを怒らせ、削除・低評価につながりやすい。

### ペイウォールモーダルの実装例

```typescript
// popup.tsx — ペイウォールモーダル（ソフトブロック型）
function PaywallModal({ onClose }: { onClose: () => void }) {
  const handleUpgrade = () => {
    chrome.tabs.create({
      url: 'https://extensionpay.com/extension/your-extension-id',
    });
  };

  return (
    <div className="paywall-overlay">
      <div className="paywall-modal">
        <button className="close-btn" onClick={onClose}>✕</button>

        <div className="paywall-content">
          <h2>Pro版で制限を解除</h2>
          <p>毎月のガイド保存数の上限に達しました。</p>

          <ul className="pro-features">
            <li>✓ ガイド無制限保存</li>
            <li>✓ カスタムテンプレート</li>
            <li>✓ PDFエクスポート</li>
          </ul>

          <div className="price-display">
            <span className="price">$3.99</span>
            <span className="period">/月</span>
            <span className="daily">（1日あたり約13円）</span>
          </div>

          <button className="upgrade-btn" onClick={handleUpgrade}>
            14日間無料で試す
          </button>

          <p className="cancel-note">いつでもキャンセル可能</p>
        </div>
      </div>
    </div>
  );
}
```

**重要なUI要素:**
- **閉じるボタンは必須**: ✕ボタンを隠すとレビューで怒られます
- **価格は小さく**: 機能のベネフィットを先に、価格は後で
- **1日あたり換算**: 「$3.99/月」より「1日13円」の方が心理的に安く感じる
- **キャンセルの明示**: 「いつでも解約可能」を目立つ位置に書く

## 退会・返金対応

ExtensionPayを使っている場合、サブスクリプションの管理はExtensionPayのダッシュボードとStripeが担います。

### 返金要求への対応テンプレート

```
件名: Re: Refund Request — [拡張名]

Hi [Name],

Thank you for reaching out. I've processed a full refund for your
subscription. It should appear within 5-10 business days.

I'm sorry the extension didn't meet your needs. If you'd like to
share what wasn't working, I'm always looking to improve.

[Developer Name]
```

**方針**: 金額を問わず、要求があれば即返金します。Stripeの返金手数料（数十円）より、低評価レビューの方が長期的なダメージが大きい。

## 月額 vs 買い切りの判断軸

| 軸 | 月額サブスク向き | 買い切り向き |
|---|-------------|----------|
| 使用頻度 | 毎日使う | たまに使う |
| 機能更新 | 継続的に機能追加 | 完成した静的ツール |
| LTV目標 | 長期収益最大化 | 即時回収 |
| ユーザー心理 | SaaSに慣れた層 | 「月額は嫌」な層 |

### LTV比較

```
月額 $3.99/月:
  平均継続月数 = 8ヶ月（S-Hub実測）
  LTV = $3.99 × 8 = $31.92

買い切り $14.99:
  即時LTV = $14.99
  → 平均継続4ヶ月未満なら買い切りがLTV高い

# 現実的な判断
- 毎日使う系（FocusGuard）→ 月額
- 作業ツール系（Procshot）→ 月額
- 調べ物系（JFF）→ 買い切り or 無料
- ライトユース系（ReadMark）→ 買い切り ($4.99) を選択
```

ReadMarkを買い切り$4.99に設定した理由は、「スクロール位置を保存したい」というニーズが「毎日使うが意識しない」型であり、月額課金への心理的抵抗感が強いユーザー層だと判断したためです。
