「パスワードをMD5でハッシュ化している」——これは危険です。ハッシュ関数の種類と使い分けを正しく理解しましょう。


[](#%E3%83%8F%E3%83%83%E3%82%B7%E3%83%A5%E9%96%A2%E6%95%B0%E3%81%A8%E3%81%AF) ハッシュ関数とは

ハッシュ関数は、**任意のデータを固定長の文字列（ハッシュ値）に変換する関数**です。


```
"Hello" → SHA-256 → 185f8db32921bd46d35...（64文字の16進数）
"hello" → SHA-256 → 2cf24dba5fb0a30e26e...（まったく違うハッシュ値）
```

ハッシュ関数の重要な性質：


- 同じ入力からは必ず同じ出力

- わずかな変化で出力が大きく変わる（雪崩効果）

- 出力から入力を逆算できない（一方向性）

- 異なる入力で同じ出力が生じにくい（衝突耐性）

[](#%E3%83%91%E3%82%B9%E3%83%AF%E3%83%BC%E3%83%89%E4%BF%9D%E5%AD%98%E3%81%AB%E3%81%AF%E7%B5%B6%E5%AF%BE%E3%81%ABmd5%2Fsha-256%E3%82%92%E4%BD%BF%E3%82%8F%E3%81%AA%E3%81%84) パスワード保存には絶対にMD5/SHA-256を使わない

MD5でパスワードをハッシュ化することは**非常に危険**です。理由は2つ：


**レインボーテーブル攻撃**：よくあるパスワードのハッシュ値をあらかじめ計算したテーブルを使って逆引きできる

**高速すぎる**：GPUを使えば1秒間に数十億回の試行が可能


パスワードの保存には**bcrypt・Argon2・scrypt**のような専用ハッシュ関数を使ってください。


```
// ❌ 絶対にやってはいけない
const hashedPassword = md5(password);
 
// ✅ bcryptを使う
import bcrypt from "bcrypt";
const hashedPassword = await bcrypt.hash(password, 10); // コスト係数=10
const isValid = await bcrypt.compare(password, hashedPassword);
```

**ファイルの完全性確認（チェックサム）**

ダウンロードしたファイルが改ざんされていないか確認します。


```
sha256sum downloaded_file.zip
# 公式サイトのSHA-256値と比較
```

**デジタル署名**

メッセージのハッシュ値に秘密鍵で署名することで、内容が改ざんされていないことを証明します。


**コンテンツアドレッシング**

GitはコミットやファイルをSHA-1ハッシュで管理しています。


[](#web-crypto-api%E3%81%A7%E3%81%AE%E5%AE%9F%E8%A3%85) Web Crypto APIでの実装

```
async function sha256(message) {
  const msgBuffer = new TextEncoder().encode(message);
  const hashBuffer = await crypto.subtle.digest("SHA-256", msgBuffer);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map((b) => b.toString(16).padStart(2, "0")).join("");
}
 
const hash = await sha256("Hello, World!");
console.log(hash);
// "dffd6021bb2bd5b0af676290809ec3a53191dd81c7f70a4b28688a362182986d"
```
