**Manifest V3への移行は思ったより大変だった**


2023年にChrome拡張のManifest V2がサポート終了し、V3への移行が必要になりました。


最も苦労したのはService Workerの扱いです。V2のBackground Scriptは常駐していましたが、V3のService Workerはアイドル時に終了してしまいます。


```
// V3で状態を保持するパターン
// Service Workerは再起動されるので、すべての状態をchrome.storage.localに保存する
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  // ExtPayも含め、すべてのコールバック内で再宣言が必要
  const extpay = ExtPay("extension-id");
  extpay.getUser().then(user => {
    sendResponse({ isPaid: user.paid });
  });
  return true; // 非同期レスポンスのためにtrueを返す
});
```

**Shadow DOMはコンテンツスクリプトの必須知識**


Webページ上にUIを注入する際、ページのCSSと競合しないようにShadow DOMを使います。


```
const host = document.createElement("div");
document.body.appendChild(host);
const shadow = host.attachShadow({ mode: "open" });
// shadow内はページのCSSが当たらない
```

**作っても誰も来ない**


最初の5本くらいは「良いものを作れば人が来る」と思っていました。全然来ませんでした。


Chrome Web Store（CWS）のオーガニック検索でユーザーを獲得するには、タイトルとディスクリプションのASO（App Store Optimization）が重要です。


**タイトルキーワードが最重要**


CWSではタイトルに含まれるキーワードが検索順位に最も影響します。競合の少ないキーワードでニッチ1位を狙う方が、激戦区の10位より遥かに効果的です。


**Zennの記事は最高の集客媒体だった**


「この拡張機能を作りました」的な記事を1本書いたら1,000PVを達成し、その記事経由でのインストール数が他のすべての経路を合わせたより多かったです。


[](#%E5%8F%8E%E7%9B%8A%E5%8C%96%E3%81%A7%E5%AD%A6%E3%82%93%E3%81%A0%E3%81%93%E3%81%A8) 収益化で学んだこと

**ExtPayは神ツール**


個人開発者向けのStripe決済ライブラリExtPayを使っています。サーバー不要でChrome拡張に決済機能を追加できます。


有料化した拡張（DataPick・$19買い切り）は、無料の拡張17本を合わせたよりも収益が高いです。


**無料拡張のコスト**


ユーザーサポートのコストを甘く見ていました。無料でも「動かない」「使い方がわからない」という問い合わせが来ます。有料の方がユーザーが真剣に使ってくれるのでサポートコストが下がります。
