[](#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB) はじめに

技術記事やニュースをブラウザで読んでいるとき、本文に集中できないことはないだろうか。サイドバーの広告、ナビゲーションメニュー、関連記事のレコメンド、SNSシェアボタン。読みたいのは本文だけなのに、視界にはノイズが多すぎる。


SafariやFirefoxにはネイティブのリーダーモードが搭載されている。しかし、世界シェアトップのChromeには、長い間ネイティブで安定したリーダーモードが提供されていない。flags で実験的機能として触れることはできるが、常用するには不安定で機能も限定的だ。


ないなら作ろう。そう考えてChrome拡張機能「ZenRead」を開発した。


**ZenRead - Chrome Web Store:**
[https://chromewebstore.google.com/detail/zenread/adoiakplckmoahmiainobfpmdomnhomj](https://chromewebstore.google.com/detail/zenread/adoiakplckmoahmiainobfpmdomnhomj)


[](#zenread%E3%81%A7%E3%81%A7%E3%81%8D%E3%82%8B%E3%81%93%E3%81%A8) ZenReadでできること

[](#%E3%83%AA%E3%83%BC%E3%83%80%E3%83%BC%E3%83%A2%E3%83%BC%E3%83%89%EF%BC%88%E7%84%A1%E6%96%99%EF%BC%89) リーダーモード（無料）

ZenReadのコア機能はリーダーモードだ。任意のWebページでアイコンをクリック、またはキーボードショートカットを押すと、ページ全体が本文だけのクリーンな表示に切り替わる。


表示のカスタマイズは細かく設定できる。


**テーマ**: Light / Dark / Sepia の3種類

**フォントサイズ**: S（16px）/ M（18px）/ L（20px）/ XL（24px）の4段階

**フォントファミリー**: Sans-serif / Serif / Monospace の3種類

**コンテンツ幅**: Narrow（600px）/ Medium（720px）/ Wide（900px）

**行間**: 1.4 / 1.6 / 1.8 / 2.0 の4段階


設定は `chrome.storage.local` に保存されるため、次回以降も同じ設定で使える。


[](#%E6%8E%A8%E5%AE%9A%E8%AA%AD%E4%BA%86%E6%99%82%E9%96%93) 推定読了時間

記事を開くと、本文の文字数から推定読了時間が自動計算される。英語は265 WPM、日本語は500文字/分を基準にしており、言語に応じて切り替わる仕組みだ。


[](#%E3%82%A2%E3%82%AF%E3%82%BB%E3%82%B7%E3%83%93%E3%83%AA%E3%83%86%E3%82%A3%E6%A9%9F%E8%83%BD%EF%BC%88%E7%84%A1%E6%96%99%EF%BC%89) アクセシビリティ機能（無料）

ZenReadにはいくつかのアクセシビリティ支援機能が含まれている。


**Focus Bold（バイオニックリーディング）** は、各単語の前半部分を太字にすることで、視線の流れを誘導し、読み速度の向上を助ける手法だ。太字にする比率はスライダーで30%〜70%の範囲で調整できる。Alt+B でオン/オフを切り替えられる。


**ディスレクシアフォント** として、OpenDyslexic・Lexie Readable・Comic Neue に切り替え可能。読字障害がある方だけでなく、長時間の読書で目が疲れやすい方にも有用だ。


**カラーオーバーレイ** は、画面全体に薄い色のフィルタをかける機能。Yellow / Blue / Pink / Green / Peach / Lavender の6色から選べる。光過敏やアーレンシンドロームなどの症状がある方の読書負担を軽減する目的で実装した。


**リーディングルーラー** は、現在読んでいる行をハイライト表示するガイド線機能。行を見失いがちな方の補助として機能する。Alt+L で切り替え可能。


**文字間隔・単語間隔の調整** も用意しており、Normal / Wide / Wider / Widest の4段階から選択できる。


[](#%E3%83%86%E3%82%AD%E3%82%B9%E3%83%88%E8%AA%AD%E3%81%BF%E4%B8%8A%E3%81%92---tts%EF%BC%88pro%EF%BC%89) テキスト読み上げ - TTS（Pro）

Web Speech APIを利用した読み上げ機能。記事の本文を音声で聴くことができる。


技術的な工夫として、以下の処理を行っている。


**Chrome 14秒バグの回避**: Chromeには、SpeechSynthesisが約14秒で停止するバグがある。10秒ごとに `pause()` → `resume()` を呼ぶワークアラウンドを実装している

**テキストの前処理**: TTSに渡す前に、URLを「リンク」に、メールアドレスを「メールアドレス」に置換する。日本語の場合は「AI」を「エーアイ」、「API」を「エーピーアイ」のように略語をカタカナ読みに変換する

**テキストのチャンク分割**: 長文を文の区切りで適切に分割し、自然な間で読み上げる

**音声の自動選択**: 言語を自動検出し、OSにインストールされた音声の中から品質の高いものを優先的に選択する。Microsoft Nanami（日本語）やGoogle US English（英語）などを優先する

**再生速度**: 0.75x / 1.0x / 1.25x / 1.5x / 2.0x の5段階


通勤中や家事をしながら技術記事を「聴く」といった使い方を想定している。


[](#%E3%83%8F%E3%82%A4%E3%83%A9%E3%82%A4%E3%83%88%E3%83%BB%E3%83%A1%E3%83%A2%EF%BC%88pro%EF%BC%89) ハイライト・メモ（Pro）

本文中のテキストを選択すると、5色（Yellow / Green / Blue / Pink / Purple）のハイライトを付けられる。ハイライトにメモを添付することも可能で、記事を読みながらの情報整理に使える。


[](#%E3%82%A8%E3%82%AF%E3%82%B9%E3%83%9D%E3%83%BC%E3%83%88%EF%BC%88pro%EF%BC%89) エクスポート（Pro）

リーダーモードで表示中の記事を、Markdown / Plain Text / HTML の3形式でダウンロードできる。クリップボードへのコピーにも対応している。HTML→Markdown変換はDOM操作による独自実装で、見出し・リスト・コードブロック・画像などの構造を保持する。


[](#%E3%81%9D%E3%81%AE%E4%BB%96%E3%81%AE%E6%A9%9F%E8%83%BD) その他の機能


**連続読書ストリーク**: 毎日記事を読むとストリークがカウントされる。ゲーミフィケーション要素として実装

**印刷最適化**: リーダーモードの表示をそのまま印刷可能。コントロールバーや設定パネルは自動的に非表示になる

**共有**: Web Share APIに対応。記事のURLをZenReadのパラメータ付きで共有できる

**多言語対応**: 日本語・英語・韓国語・フランス語・ドイツ語・スペイン語・イタリア語・ポルトガル語（ブラジル）の8言語に対応

[](#readability.js-%E3%81%AB%E3%82%88%E3%82%8B%E8%A8%98%E4%BA%8B%E6%8A%BD%E5%87%BA) Readability.js による記事抽出

記事本文の抽出にはMozillaが開発している `@mozilla/readability` を使っている。これはFirefoxのリーダーモードでも使われているライブラリだ。


```
const clone = doc.cloneNode(true) as Document;
const reader = new Readability(clone, {
  charThreshold: 500,
  keepClasses: false,
  disableJSONLD: false,
});
const article = reader.parse();
```

ポイントは `doc.cloneNode(true)` で文書のクローンを作成してから解析していること。Readabilityは解析時にDOMを変更するため、元のページを壊さないための対策だ。


抽出後のHTMLはDOMPurifyでサニタイズし、`script`, `style`, `iframe`, `form` などのタグや、`onerror`, `onclick` などの属性を除去している。


[](#%E3%82%B5%E3%82%A4%E3%83%88%E5%9B%BA%E6%9C%89%E3%83%AB%E3%83%BC%E3%83%AB%E3%81%AB%E3%82%88%E3%82%8B%E6%8A%BD%E5%87%BA%E7%B2%BE%E5%BA%A6%E3%81%AE%E5%90%91%E4%B8%8A) サイト固有ルールによる抽出精度の向上

汎用のReadabilityだけでは抽出がうまくいかないサイトもある。そのため、主要なサイトに対してはカスタムのCSSセレクタルールを定義している。


```
const siteRules: Recordstring, SiteRule> = {
  'qiita.com': {
    contentSelector: '.it-MdContent',
    titleSelector: 'h1.it-Header_title',
    authorSelector: '.it-Header_authorName',
    preRemove: ['.it-Footer', '.it-Sidebar', '.it-RelatedArticles']
  },
  'zenn.dev': {
    contentSelector: '.znc',
    titleSelector: 'h1',
    preRemove: ['.Sidebar', '.ArticleFooter']
  },
  // ... Medium, Wikipedia, note.com, NY Times なども対応
};
```

`preRemove` で不要な要素を事前に削除してからReadabilityに渡すことで、抽出精度を上げている。


[](#%E3%82%B3%E3%83%B3%E3%83%86%E3%83%B3%E3%83%84%E3%82%B9%E3%82%AF%E3%83%AA%E3%83%97%E3%83%88%E3%81%AE%E8%A8%AD%E8%A8%88) コンテンツスクリプトの設計

Chrome拡張のコンテンツスクリプトでReactを動かすには少し工夫が必要だ。ZenReadでは以下のアプローチを取っている。


- ページのDOMに `#zenread-root` を追加

- そこに `createRoot` でReactをマウント

- スタイルは `` タグとして `document.head` に注入

- z-indexを `2147483647`（32bit整数の最大値）に設定し、既存のページ要素の上にオーバーレイ


コンテンツスクリプトはIIFE形式で別途esbuildでビルドし、Viteのポップアップ/ウェルカムページ用ビルドとは分離している。

[](#%E3%81%8A%E3%82%8F%E3%82%8A%E3%81%AB) おわりに

ZenReadは、Webで長文を読む体験を改善するために作ったChrome拡張機能だ。Readability.jsによる記事抽出、Focus Boldによる速読支援、Web Speech APIによるTTS、アクセシビリティ機能など、読書に特化した機能を詰め込んでいる。
