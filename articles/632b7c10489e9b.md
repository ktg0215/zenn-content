[](#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB) はじめに

技術記事やドキュメントを読んでいて、途中で離脱した後に戻ってきたら「どこまで読んだっけ？」となった経験はありませんか？


長いQiitaの記事、公式ドキュメント、ブログ記事…。スクロール位置を覚えていられるほど人間の記憶は正確ではありません。


この問題を解決するために、**ReadMark**というChrome拡張を作りました。


[](#readmark%E3%81%A8%E3%81%AF) ReadMarkとは

ReadMarkは、Webページのスクロール位置を自動的に保存し、再訪問時にワンクリックで復元できるChrome拡張です。


主な機能はシンプルです。


**自動保存**: ページを読んでいるだけでスクロール位置を自動記録

**ワンクリック復元**: 再訪問時にボタン一つで前回の位置に戻れる

**複数ページ管理**: 複数のページの読書位置を同時に管理

**ビジュアルマーカー**: 前回の位置が視覚的にわかる


[](#%E3%81%AA%E3%81%9C%E4%BD%9C%E3%81%A3%E3%81%9F%E3%81%8B) なぜ作ったか

個人的に技術ドキュメントを読む機会が多く、特にReactやTypeScriptの公式ドキュメントは長大です。「昨日どこまで読んだっけ？」と毎回最初からスクロールし直すのが地味にストレスでした。


ブラウザのブックマーク機能ではページ単位でしか保存できず、ページ内の位置までは記録されません。この「ちょっとした不便」を解消したくて作りました。


[](#%E6%8A%80%E8%A1%93%E6%A7%8B%E6%88%90) 技術構成

```
TypeScript + React
Vite + CRXJS（ビルド）
Tailwind CSS（UI）
Chrome Storage API（データ保存）
```

[](#%E4%BB%95%E7%B5%84%E3%81%BF) 仕組み

ReadMarkの動作はシンプルです。


**Content Script**がページのスクロールイベントを監視

- スクロールが停止したタイミングで位置を**Chrome Storage**に保存

- 同じページに再訪問すると、保存された位置を検出
ポップアップから**ワンクリックで復元**


データはすべてローカルに保存されるため、プライバシーの心配もありません。


[](#%E4%BD%BF%E3%81%84%E6%96%B9) 使い方

インストールするだけで使えます。特別な設定は不要です。


- Chrome Web Storeからインストール

- 普通にWebページを読む（自動で位置が記録される）

- 後日同じページを開いたら、拡張アイコンをクリック

- 「前回の位置に戻る」で一発復元


[](#%E3%81%93%E3%82%93%E3%81%AA%E4%BA%BA%E3%81%AB%E3%81%8A%E3%81%99%E3%81%99%E3%82%81) こんな人におすすめ


- 技術記事を日常的に読むエンジニア

- 長い公式ドキュメントを少しずつ読み進めている方

- 調査やリサーチで複数の長文ページを並行して読む方

[](#%E3%81%95%E3%81%84%E3%81%94%E3%81%AB) さいごに

ReadMarkは「あったら便利だけど、なくても困らない」タイプのツールです。でも一度使い始めると、なかったときには戻れなくなります。


地味ですが、日々の読書体験を確実に改善してくれる拡張です。ぜひ試してみてください。
