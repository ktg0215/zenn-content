Webアプリの認証で頻繁に使われるJWT（JSON Web Token）。「何となく使っている」という方に向けて、構造から安全な使い方まで解説します。


[](#jwt%E3%81%A8%E3%81%AF) JWTとは

JWT（ジェイソン・ウェブ・トークン）は、**JSON形式のデータを安全に送受信するためのコンパクトなトークン形式**です。


主にWebアプリの認証・認可に使われます。


[](#jwt%E3%81%AE%E6%A7%8B%E9%80%A0) JWTの構造

JWTは3つのパートをピリオド（`.`）でつないだ文字列です。


```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOiIxMjMiLCJleHAiOjE3MDAwMDAwMDB9.xxxxx
```

**① ヘッダー（Header）**

アルゴリズムとトークンタイプを含みます。


```
{
  "alg": "HS256",
  "typ": "JWT"
}
```

**② ペイロード（Payload）**

実際のデータ（クレーム）を含みます。


```
{
  "userId": "123",
  "email": "user@example.com",
  "exp": 1700000000,
  "iat": 1699996400
}
```

よく使われる標準クレーム：


`sub`：トークンの主体（ユーザーIDなど）

`exp`：有効期限（Unix時間）

`iat`：発行日時

`iss`：発行者


**③ 署名（Signature）**

ヘッダーとペイロードを秘密鍵でハッシュ化したものです。


```
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)
```

[](#%E8%AA%8D%E8%A8%BC%E3%83%95%E3%83%AD%E3%83%BC) 認証フロー

```
1. ユーザーがIDとパスワードでログイン
        ↓
2. サーバーがJWTを生成して返す
        ↓
3. クライアントがJWTを保存（localStorage or Cookie）
        ↓
4. 次回リクエスト時にJWTをAuthorizationヘッダーに付与
        Authorization: Bearer eyJhbGci...
        ↓
5. サーバーがJWTの署名を検証
        ↓
6. 有効ならリクエストを処理
```

[](#%E3%82%BB%E3%82%AD%E3%83%A5%E3%83%AA%E3%83%86%E3%82%A3%E3%81%AE%E8%90%BD%E3%81%A8%E3%81%97%E7%A9%B4) セキュリティの落とし穴

**1. alg: "none" 攻撃**

アルゴリズムに"none"を指定した改ざんJWTを受け入れてしまう脆弱性があります。必ず使用するアルゴリズムを明示的に指定してください。


```
// 危険：algを検証しない
jwt.verify(token, secret);
 
// 安全：algを明示
jwt.verify(token, secret, { algorithms: ["HS256"] });
```

**2. ペイロードは暗号化されていない**

Base64Urlエンコードされているだけなので、誰でもデコードできます。秘密情報（パスワード、クレジットカード番号など）はペイロードに含めないでください。


**3. 保存場所の問題**


- localStorage：XSS攻撃でトークンが盗まれるリスク

- Cookie（HttpOnly）：XSSには強いが、CSRF対策が必要


[](#javascript%E3%81%A7%E3%81%AE%E3%83%87%E3%82%B3%E3%83%BC%E3%83%89) JavaScriptでのデコード

署名の検証なしにペイロードを確認したい場合（デバッグ目的など）：


```
function decodeJWT(token) {
  const [header, payload] = token.split(".");
  
  const decodeBase64Url = (str) => {
    const pad = str.length % 4;
    const padded = str + "=".repeat(pad ? 4 - pad : 0);
    return JSON.parse(atob(padded.replace(/-/g, "+").replace(/_/g, "/")));
  };
  
  return {
    header: decodeBase64Url(header),
    payload: decodeBase64Url(payload),
  };
}
```
!

署名の検証なしにJWTの内容を信頼することは絶対にやめてください。デコードはデバッグ目的のみに使用し、本番環境では必ず署名を検証してください。
