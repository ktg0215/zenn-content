[](#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB) はじめに

Chrome拡張を作っても、誰にも知られなければ意味がない。


でも毎回手動で告知するのは面倒。リリースするたびにDiscord、Bluesky、Dev.to、Qiita... 全部に投稿なんてやってられない。


そこで、**GitHub Releaseを作成したら自動で複数プラットフォームに告知**されるシステムをn8nで構築しました。


[](#%E5%AE%8C%E6%88%90%E3%81%97%E3%81%9F%E3%82%B7%E3%82%B9%E3%83%86%E3%83%A0) 完成したシステム

```
GitHub Release作成
    ↓ Webhook
n8n が検知
    ↓
自動で投稿
├── Discord
├── Bluesky  
├── Dev.to
└── Qiita
```

一度リリースを作れば、4つのプラットフォームに自動投稿されます。


[](#%E4%BD%BF%E3%81%A3%E3%81%9F%E6%8A%80%E8%A1%93) 使った技術


コンポーネント
役割

リリース作成を検知


Cloudflare Tunnel
ローカルn8nを外部公開


n8n
ワークフロー管理


テンプレート
各プラットフォーム用の投稿文


[](#%E6%A7%8B%E7%AF%89%E6%89%8B%E9%A0%86) 構築手順

[](#1.-n8n%E3%82%92%E3%83%AD%E3%83%BC%E3%82%AB%E3%83%AB%E3%81%A7%E8%B5%B7%E5%8B%95) 1. n8nをローカルで起動

Dockerで起動するのが一番簡単です。


```
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

[](#2.-cloudflare-tunnel%E3%81%A7n8n%E3%82%92%E5%85%AC%E9%96%8B) 2. Cloudflare Tunnelでn8nを公開

ローカルのn8nをGitHub Webhookから叩けるようにします。


```
# cloudflaredをインストール
brew install cloudflare/cloudflare/cloudflared

# トンネル作成
cloudflared tunnel create n8n-tunnel

# 設定ファイル作成（~/.cloudflared/config.yml）
```

```
tunnel: [トンネルID]
credentials-file: /Users/[ユーザー名]/.cloudflared/[トンネルID].json

ingress:
  - hostname: n8n.yourdomain.com
    service: http://localhost:5678
  - service: http_status:404
```

```
# DNS設定
cloudflared tunnel route dns n8n-tunnel n8n.yourdomain.com

# 起動
cloudflared tunnel run n8n-tunnel
```

[](#3.-github-webhook%E3%82%92%E8%A8%AD%E5%AE%9A) 3. GitHub Webhookを設定

リポジトリの Settings → Webhooks → Add webhook


項目
設定値


Payload URL
[https://n8n.yourdomain.com/webhook/github-release](https://n8n.yourdomain.com/webhook/github-release)


Content type
application/json


Events
Releases のみ


[](#4.-n8n%E3%81%A7%E3%83%AF%E3%83%BC%E3%82%AF%E3%83%95%E3%83%AD%E3%83%BC%E4%BD%9C%E6%88%90) 4. n8nでワークフロー作成

[](#%E3%83%8E%E3%83%BC%E3%83%89%E6%A7%8B%E6%88%90) ノード構成

```
Webhook（トリガー）
    ↓
IF（releaseイベントか判定）
    ↓
Set（投稿内容を整形）
    ↓
├── Discord（Webhook）
├── Bluesky（HTTP Request）
├── Dev.to（HTTP Request）
└── Qiita（HTTP Request）
```

[](#%E6%8A%95%E7%A8%BF%E3%83%86%E3%83%B3%E3%83%97%E3%83%AC%E3%83%BC%E3%83%88%E4%BE%8B) 投稿テンプレート例

```
// Setノードで設定
const releaseName = $json.release.name;
const releaseUrl = $json.release.html_url;
const repoName = $json.repository.name;

return {
  discord: `🚀 **${repoName}** の新バージョンをリリースしました！\n${releaseName}\n${releaseUrl}`,
  bluesky: `🚀 ${repoName} の新バージョンをリリース！\n${releaseName}\n${releaseUrl}`,
  devto: {
    title: `${repoName} ${releaseName} Released`,
    body: `New version available: ${releaseUrl}`
  }
};
```


[](#%E3%83%8F%E3%83%9E%E3%81%A3%E3%81%9F%E3%83%9D%E3%82%A4%E3%83%B3%E3%83%88) ハマったポイント

[](#openai-api-%E2%86%92-%E3%83%86%E3%83%B3%E3%83%97%E3%83%AC%E3%83%BC%E3%83%88%E3%81%AB%E5%A4%89%E6%9B%B4) OpenAI API → テンプレートに変更

最初はOpenAI APIで投稿文を動的生成しようとしました。


**問題点:**


- APIの応答が不安定

- たまに変な文章が生成される

- レート制限に引っかかる


**解決策:**

テンプレートベースに切り替え。シンプルで確実。


[](#cloudflare-tunnel%E3%81%AE%E8%A8%AD%E5%AE%9A) Cloudflare Tunnelの設定

設定に半日かかりました。ポイントは：


`config.yml`の場所を間違えない

- DNSの反映に時間がかかることがある

- ローカルでn8nが起動していることを確認

[](#%E3%81%BE%E3%81%A8%E3%82%81) まとめ


- n8n + Cloudflare Tunnelで無料で自動化できる

- OpenAI APIより テンプレートの方が安定

- X/TwitterはAPI料金が高すぎて断念

- 全部自動化せず、手動の方が効果的な部分は残す


一度構築すれば、リリースのたびに自動で告知されるので楽です。
