この記事では、プログラミング初心者がAIを活用してChrome拡張を開発する具体的なフローを紹介します。

ポイントは**Claudeだけ有料版**にしていること。設計と指示書作成が一番重要なので、ここに投資しています。


[](#%E9%96%8B%E7%99%BA%E3%83%95%E3%83%AD%E3%83%BC) 開発フロー

[](#%E5%85%A8%E4%BD%93%E3%81%AE%E6%B5%81%E3%82%8C) 全体の流れ

```
① AIでリサーチ（収益化できそうなアプリを探す）
    ↓
② Claudeで内容をまとめてランキング形式にする
    ↓
③ 1位から開発開始
    ↓
④ Claudeで開発の指示書を作成
    ↓
⑤ Claude Code に投げる
    ↓
⑥ 完成
```

[](#%E2%91%A0-%E3%83%AA%E3%82%B5%E3%83%BC%E3%83%81) ① リサーチ

Claude、ChatGPT、Geminiの3つを使ってリサーチします。


**AIによってリサーチ結果が結構違う**ので、複数使うのがおすすめ。


```
プロンプト例：
「Chrome拡張機能で収益化できそうなアイデアを10個出して。
 競合が少なく、個人開発で作れる規模感のもの」
```

[](#%E2%91%A1-%E3%83%A9%E3%83%B3%E3%82%AD%E3%83%B3%E3%82%B0%E5%8C%96) ② ランキング化

リサーチ結果をClaudeに渡して、ランキング形式にまとめてもらいます。


```
プロンプト例：
「以下のアイデアを、収益性・開発難易度・競合状況を考慮して
 ランキング形式にまとめて」
```

[](#%E2%91%A2-%E6%8C%87%E7%A4%BA%E6%9B%B8%E4%BD%9C%E6%88%90) ③ 指示書作成

開発するアプリが決まったら、Claudeに指示書を作ってもらいます。


```
プロンプト例：
「○○というChrome拡張を作りたい。
 以下を含む開発指示書を作成して：
 - ファイル構成
 - 各ファイルの役割
 - 実装手順
 - manifest.jsonの内容」
```

[](#%E2%91%A3-%E5%AE%9F%E8%A3%85) ④ 実装

指示書をClaude Codeに渡して、実装してもらいます。


自分が書くより、AIに任せた方が確実に質が高い。そして圧倒的に早い。


[](#%E2%91%A4-%E3%83%87%E3%83%90%E3%83%83%E3%82%B0) ⑤ デバッグ

動かない場合はエラーをそのままClaudeに貼って相談。


```
プロンプト例：
「以下のエラーが出ています。原因と解決方法を教えて：
 [エラーメッセージをコピペ]」
```

覚えるべきは **「何ができるのか」** を知ること。

[](#ai%E3%81%B8%E3%81%AE%E5%90%91%E3%81%8D%E5%90%88%E3%81%84%E6%96%B9) AIへの向き合い方

[](#%E5%9F%BA%E6%9C%AC%E3%82%B9%E3%82%BF%E3%83%B3%E3%82%B9) 基本スタンス


- 分からないことは全部AIに聞く

- おすすめに素直に乗っかる

- 慣れてくると疑問も出てくる → また質問する


この繰り返しで十分やっていける。

[](#%E6%B3%A8%E6%84%8F%E7%82%B9) 注意点

AIを複数同時に動かしたりもしていますが、PCスペック次第では普通に落ちます。


[](#%E5%8F%8E%E7%9B%8A%E5%8C%96%EF%BC%9Aextensionpay) 収益化：ExtensionPay

Chrome拡張の収益化にはExtensionPayを使っています。


[](#%E9%81%B8%E3%82%93%E3%81%A0%E7%90%86%E7%94%B1) 選んだ理由


- Stripe連携が簡単

- 買い切り / サブスク両対応

- 実装が数行で済む


[](#%E3%81%BE%E3%81%A8%E3%82%81) まとめ


- コード書けなくてもAIで開発できる

- 重要なのは「指示の出し方」

- 複数AIを使い分けるとリサーチの質が上がる

- ExtensionPayで収益化も簡単
