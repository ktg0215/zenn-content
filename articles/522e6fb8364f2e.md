「タイムスタンプが9桁？10桁？13桁？」と混乱したことはありませんか？UNIXタイムスタンプの基礎から実務での注意点まで解説します。


[](#unix%E3%82%BF%E3%82%A4%E3%83%A0%E3%82%B9%E3%82%BF%E3%83%B3%E3%83%97%E3%81%A8%E3%81%AF) UNIXタイムスタンプとは

UNIXタイムスタンプは、**1970年1月1日0時0分0秒（UTC）からの経過秒数**を表す整数値です。


```
2024-01-01 00:00:00 UTC = 1704067200
```

[](#%E3%81%AA%E3%81%9C%E3%82%BF%E3%82%A4%E3%83%A0%E3%82%B9%E3%82%BF%E3%83%B3%E3%83%97%E3%82%92%E4%BD%BF%E3%81%86%E3%81%AE%E3%81%8B) なぜタイムスタンプを使うのか

**タイムゾーンの問題を回避できる**

日時データには必ずタイムゾーンの問題がついてきます。タイムスタンプはタイムゾーンに依存しないため、異なるタイムゾーンのシステム間でデータをやり取りする際に信頼性が高いです。


**比較・計算が簡単**

整数値なので、「どちらが新しいか」「経過時間は何秒か」を単純な数値比較・減算で求められます。


```
const diff = timestamp2 - timestamp1; // 経過秒数
```

[](#%E7%A7%92-vs-%E3%83%9F%E3%83%AA%E7%A7%92%E3%81%AE%E6%B7%B7%E4%B9%B1) 秒 vs ミリ秒の混乱

**最もよくあるバグの原因**です。


**UNIXタイムスタンプ（秒）**：10桁 → `1704067200`


**JavaScriptのDate**（ミリ秒）：13桁 → `1704067200000`


```
// JavaScriptのDate.now()はミリ秒
Date.now(); // 1704067200000（13桁）
 
// サーバーからの秒単位タイムスタンプをDateに変換
const unixSeconds = 1704067200;
const date = new Date(unixSeconds * 1000); // ← ×1000 を忘れずに
```

[](#javascript%E3%81%A7%E3%81%AE%E5%A4%89%E6%8F%9B) JavaScriptでの変換

```
// 現在のUNIXタイムスタンプ（秒）
const nowSeconds = Math.floor(Date.now() / 1000);
 
// タイムスタンプ → 日時文字列
const timestamp = 1704067200;
const date = new Date(timestamp * 1000);
console.log(date.toISOString()); // "2024-01-01T00:00:00.000Z"
 
// 日時 → タイムスタンプ
const dateStr = "2024-01-01T00:00:00Z";
const ts = Math.floor(new Date(dateStr).getTime() / 1000);
console.log(ts); // 1704067200
 
// タイムゾーンを考慮した日本時間での表示
const jstDate = new Intl.DateTimeFormat("ja-JP", {
  timeZone: "Asia/Tokyo",
  year: "numeric",
  month: "2-digit",
  day: "2-digit",
  hour: "2-digit",
  minute: "2-digit",
  second: "2-digit",
}).format(date);
console.log(jstDate); // "2024/01/01 09:00:00"
```

[](#2038%E5%B9%B4%E5%95%8F%E9%A1%8C) 2038年問題

32ビット符号付き整数でUNIXタイムスタンプを扱うシステムは、**2038年1月19日にオーバーフロー**します。


JavaScriptのNumberは64ビット浮動小数点数なので問題ありませんが、古いシステムには注意が必要です。
