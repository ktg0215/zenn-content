「良いものを作れば自然と人が来る」——そう信じていた頃の話をします。


[](#seo%E3%82%92%E5%AD%A6%E3%82%93%E3%81%A0%E3%81%8D%E3%81%A3%E3%81%8B%E3%81%91) SEOを学んだきっかけ

Chrome拡張を10本以上作っても、月間アクティブユーザーが数十人から増えませんでした。


転機は技術記事をZennに投稿したとき。1本の記事が1,000PVを超え、そこからの流入が劇的に増えました。「コンテンツが大事なんだ」と気づいた瞬間でした。


そしてWebサービスを作るなら、Google検索からの流入を本気で狙わないともったいないと考え直しました。


[](#%E5%80%8B%E4%BA%BA%E9%96%8B%E7%99%BA%E8%80%85%E3%81%8Cseo%E3%81%A7%E5%AD%A6%E3%82%93%E3%81%A0%E3%81%93%E3%81%A8) 個人開発者がSEOで学んだこと

**①タイトルタグが全て**


Googleの検索結果に表示されるタイトルは、クリック率を大きく左右します。


```
title>Dev Tools Hubtitle>
 

title>JSONフォーマッター | JSON整形・beautify ツール - Dev Tools Hubtitle>
```

**②検索意図を理解する**


「JSON 整形」で検索するユーザーが本当に欲しいのは何か？説明文ではなく、**動くツール**です。ツールをページの最上部（ファーストビュー）に配置することで、ユーザーの検索意図を即座に満たせます。


**③ロングテールキーワードを狙う**


「JSON」という単語は競合が多すぎます。代わりに「JSON 整形 オンライン」「JSON フォーマッター コピー」のようなロングテールキーワードを複数ページで対策することで、少ないリソースで着実に流入を増やせます。


**④構造化データ（JSON-LD）は必須**


ツールサイトではWebApplicationスキーマを実装すると、検索結果でのリッチスニペット表示の可能性が上がります。


```
{
  "@context": "https://schema.org",
  "@type": "WebApplication",
  "name": "JSONフォーマッター",
  "applicationCategory": "DeveloperApplication",
  "offers": {
    "@type": "Offer",
    "price": "0"
  }
}
```

**⑤被リンクは地道に**


Awesome-listへのPR、技術記事での言及、GitHubのREADMEなど、小さな被リンクを積み重ねることが大切です。


[](#seo%E3%81%AE%E7%8F%BE%E5%AE%9F) SEOの現実

正直に言うと、**SEOの効果が出るまでに3〜6ヶ月かかります**。ローンチ直後はほぼゼロです。


その間にやることは：


- ツール数を増やし続ける

- 技術記事を書き続ける

- コミュニティ（Zenn・Reddit・HN）で存在感を出す
