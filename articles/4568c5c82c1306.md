[](#vercel%E3%82%92%E3%82%84%E3%82%81%E3%81%A6cloudflare-pages%E3%81%AB%E3%81%97%E3%81%9F%E7%90%86%E7%94%B1) VercelをやめてCloudflare Pagesにした理由

最初はVercelを使おうとしましたが、**Vercelの無料プランは商用利用禁止**です。広告を掲載したり、有料サービスへ誘導するサイトには使えません。


Cloudflare Pagesは商用利用OKで帯域無制限。個人開発者に最適です。


[](#next.js-%2B-cloudflare-pages%E3%81%AE%E8%A8%AD%E5%AE%9A) Next.js + Cloudflare Pagesの設定

`next.config.ts`に`output: 'export'`を設定します。


```
// next.config.ts
const nextConfig = {
  output: "export",
  images: {
    unoptimized: true,
  },
};
```

これでビルド時に`out/`ディレクトリに静的ファイルが生成され、Cloudflare Pagesで配信できます。


GitHubにpushすると自動デプロイされます。


[](#%E5%A4%9A%E8%A8%80%E8%AA%9E%E5%AF%BE%E5%BF%9C%EF%BC%88next-intl%EF%BC%89) 多言語対応（next-intl）

SEOを考えると英語対応は必須です。`next-intl`で日英対応を実装しました。


```
/ja/json-formatter → 日本語版
/en/json-formatter → 英語版
```

ミドルウェアは`output: 'export'`と非互換なので使えません。ロケール判定はクライアントサイドで行います。

[](#%E3%82%A2%E3%83%89%E3%83%90%E3%82%A4%E3%82%B9) アドバイス

**最初から完璧にしない**

まず動くものを公開することを優先しましょう。未公開のプロジェクトは価値ゼロです。


**SEOを最初から考える**

SSGで静的HTMLを生成し、各ページに適切なtitle・descriptionを設定することを最初から意識してください。後から対応するのは思ったより手間がかかります。
