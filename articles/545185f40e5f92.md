今回は、Chrome拡張機能開発における費用面について書いていきます。当然ですが、誰でもできるだけ費用を抑えたいと考えているはずです。


[](#%E7%A7%81%E3%81%8C%E5%AE%9F%E9%9A%9B%E3%81%AB%E3%81%8B%E3%81%91%E3%81%A6%E3%81%84%E3%82%8B%E8%B2%BB%E7%94%A8) 私が実際にかけている費用

現在、私が費用をかけているのは以下の2つだけです。


項目
費用
備考


Claude Code
100ドル/月
最も大きな固定費


Chrome拡張機能の登録
5ドル
一度だけの支払い


それ以外は特に費用はかかっていません。


[](#ai%E9%96%8B%E7%99%BA%E3%83%84%E3%83%BC%E3%83%AB%E3%81%AE%E9%81%B8%E6%8A%9E%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6) AI開発ツールの選択について

AIの費用がどうしても一番高くなりますが、開発スピードによって費用は大きく変わってきます。


最初からProプランを契約する必要はなく、確か20ドルのプランもあったと思うので、まずはそのくらいの費用で始めてみるのが良いのではないでしょうか。


私はClaude Codeを使っていますが、Cursorのコンポーザーなど、他にもAIコーディングツールは数多く存在します。正直なところ、どのAIを使ってもそこまで大きな差はないと感じています。


強いて言えば、以下のような印象です：


**Claude（Code）**：プログラミングに最も適している

**ChatGPT**：様々なジャンルに対応できる万能型

**Gemini**：画像関係に強い


[](#ai%E6%90%AD%E8%BC%89%E3%82%A2%E3%83%97%E3%83%AA%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6%E3%81%AE%E8%80%83%E3%81%88) AI搭載アプリについての考え

最近、アプリにAIを搭載してリリースするのがトレンドになっているようですが、AI搭載には別途費用がかかります。そのため、私は現状では導入していません。


確かにAIを入れることで機能の幅は広がりますが、できるだけ費用を抑えたいというのが本音です。また、AIなしでも完結する機能はたくさんあります。


何より、AIを組み込んだトレンドに乗ると、競合が強すぎて勝てないのではないかと考えています。


将来的にはAIを導入したものも作ってみたいとは思っていますが、今のところはClaude Codeに「費用はゼロで抑えてください」という前提で指示を出しているため、費用がかかるものは除外して開発しています。


[](#%E3%81%BE%E3%81%A8%E3%82%81) まとめ

基本的に、Chrome拡張機能の一度だけの登録費用と、月額のAIコーディングツール費用があれば、バイブコーディングは可能です。


知識がある方であれば、AIを無料で使える範囲内で活用し、自分でコードを書くこともできるでしょう。しかし、私は基本的にコードが書けないため、その方法は取っていません。


「それならコードを勉強すればいいのでは？」と思われるかもしれません。しかし、人間がどれだけコードを勉強したところで、AIと同等のスピードでアプリ開発をするのは不可能だと思います。


であれば、コードの勉強よりも、「何ができるか」という知識を増やし、それをうまくプロンプトに落とし込む技術を磨く方が有益だというのが、現時点での私の考えです。
