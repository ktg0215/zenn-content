URLに日本語や記号を含めると文字化けする経験はありませんか？URLエンコードの仕組みを理解すれば、この問題を正しく解決できます。


[](#url%E3%82%A8%E3%83%B3%E3%82%B3%E3%83%BC%E3%83%89%E3%81%A8%E3%81%AF) URLエンコードとは

URLで使用できる文字は制限されています。日本語や一部の記号はそのままURLに含められないため、**`%XX`形式（Xは16進数）に変換**します。


これをURLエンコード（パーセントエンコーディング）と呼びます。


```
こんにちは → %E3%81%93%E3%82%93%E3%81%AB%E3%81%A1%E3%81%AF
空白(スペース) → %20 または +
& → %26
= → %3D
```

[](#url%E3%81%A7%E4%BD%BF%E3%81%88%E3%82%8B%E6%96%87%E5%AD%97%E3%83%BB%E4%BD%BF%E3%81%88%E3%81%AA%E3%81%84%E6%96%87%E5%AD%97) URLで使える文字・使えない文字

**予約なし文字（そのまま使える）**

`A-Z a-z 0-9 - _ . ~`


**予約文字（特別な意味を持つ）**

`: / ? # [ ] @ ! $ & ' ( ) * + , ; =`


これらを**データとして**URLに含める場合はエンコードが必要です。


[](#javascript%E3%81%AE3%E3%81%A4%E3%81%AE%E3%82%A8%E3%83%B3%E3%82%B3%E3%83%BC%E3%83%89%E9%96%A2%E6%95%B0) JavaScriptの3つのエンコード関数

```
// 1. encodeURIComponent
// 最も安全。すべての特殊文字をエンコード
// クエリパラメータの値に使う
encodeURIComponent("Hello, World! & Japan=日本");
// → "Hello%2C%20World!%20%26%20Japan%3D%E6%97%A5%E6%9C%AC"
 
// 2. encodeURI
// URLの構造を保ちながらエンコード
// URL全体をエンコードする場合に使う（://?#は変換しない）
encodeURI("https://example.com/検索?q=テスト");
// → "https://example.com/%E6%A4%9C%E7%B4%A2?q=%E3%83%86%E3%82%B9%E3%83%88"
 
// 3. escape (非推奨)
// 古い関数。使わないこと
```

[](#%E3%82%88%E3%81%8F%E3%81%82%E3%82%8B%E7%BD%A0) よくある罠

**encodeURIComponentとencodeURIの使い分けを間違える**


```
// クエリパラメータの値は encodeURIComponent を使う
const query = "東京 & 大阪";
const url = `https://example.com/search?q=${encodeURIComponent(query)}`;
// ✅ https://example.com/search?q=%E6%9D%B1%E4%BA%AC%20%26%20%E5%A4%A7%E9%98%AA
 
// encodeURI を使うと & がエンコードされない
const url2 = `https://example.com/search?q=${encodeURI(query)}`;
// ❌ https://example.com/search?q=%E6%9D%B1%E4%BA%AC%20&%20%E5%A4%A7%E9%98%AA
// ← & が2つ目のクエリパラメータとして解釈される
```

**+とスペースの問題**


HTMLフォームのGETメソッドでは、スペースが`%20`ではなく`+`にエンコードされることがあります。


```
// フォームデータ：スペース → +
decodeURIComponent("Hello+World"); // "Hello+World"（+は変換されない！）
 
// application/x-www-form-urlencoded形式のデコード
decodeURIComponent("Hello+World".replace(/\+/g, " ")); // "Hello World"
```
