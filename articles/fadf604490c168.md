[](#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB) はじめに

「承知しました。対応いたします。」「ご確認ありがとうございます。」「添付ファイルをお送りします。」


毎日のGmail対応で、こんな定型文を何度も手打ちしていませんか。メール1通あたりの返信時間は短くても、1日数十通ともなれば無視できない時間になります。


Gmailには「テンプレート」機能（旧・定型文）がありますが、挿入までの手順が多く、カスタマイズ性にも限界があります。この不満を解消するために、Gmail上でワンクリック・テンプレート返信ができるChrome拡張機能「SnapReply for Gmail」を開発しました。


[https://chromewebstore.google.com/detail/snapreply-for-gmail/pijandokkbhdejnoienhjmoicjapgpmk](https://chromewebstore.google.com/detail/snapreply-for-gmail/pijandokkbhdejnoienhjmoicjapgpmk)


この記事では、SnapReplyの機能紹介に加えて、Gmail拡張機能の開発で直面した技術的な課題についても触れていきます。


[](#snapreply%E3%81%A7%E3%81%A7%E3%81%8D%E3%82%8B%E3%81%93%E3%81%A8) SnapReplyでできること

[](#%E3%83%86%E3%83%B3%E3%83%97%E3%83%AC%E3%83%BC%E3%83%88%E3%82%92gmail%E4%B8%8A%E3%81%8B%E3%82%89%E7%9B%B4%E6%8E%A5%E6%8C%BF%E5%85%A5) テンプレートをGmail上から直接挿入

SnapReplyをインストールすると、Gmailの作成画面のツールバーにテンプレートボタンが追加されます。ボタンをクリックするとテンプレート一覧パネルが開き、使いたいテンプレートを選ぶだけで本文に挿入されます。


テンプレートは検索やタグで絞り込み可能で、フォルダで整理することもできます。よく使うテンプレートが上位に表示される仕組みなので、テンプレートが増えても目的のものにすぐたどり着けます。


[](#%2Fkeyword-%E3%82%B7%E3%83%A7%E3%83%BC%E3%83%88%E3%82%AB%E3%83%83%E3%83%88%E3%81%A7%E5%8D%B3%E5%BA%A7%E3%81%AB%E5%B1%95%E9%96%8B) `/keyword` ショートカットで即座に展開

テンプレートにはショートカットキーワードを設定できます。たとえば `/thanks` と登録しておけば、メール本文に `/thanks` と入力してスペースキーを押すだけで、テンプレートが展開されます。パネルを開く手間すら省けるので、本当に数秒で返信が完了します。


[](#%E5%A4%89%E6%95%B0%E3%81%A8%E3%83%86%E3%83%B3%E3%83%97%E3%83%AC%E3%83%BC%E3%83%88%E3%82%A8%E3%83%B3%E3%82%B8%E3%83%B3) 変数とテンプレートエンジン

テンプレート本文に `{{name}}` や `{{company}}` といった変数を埋め込むことができます。挿入時にプロンプトが表示され、値を入力すると自動で置換されます。前回入力した値が記憶されるため、同じ宛先への返信なら入力の手間も最小限です。


組み込み変数として `{{date}}`、`{{time}}`、`{{year}}`、`{{month}}`、`{{day}}` が用意されており、これらは自動的に現在の日時に置き換わります。日付フォーマットの指定（`{{date:YYYY/MM/DD}}`）や、条件分岐（`{{#if name}}...{{/if}}`）にも対応しています。


[](#%E4%BB%B6%E5%90%8D%E3%81%AE%E8%87%AA%E5%8B%95%E5%85%A5%E5%8A%9B) 件名の自動入力

テンプレートには件名も設定可能です。新規作成時で件名が空の場合に自動で入力されます。変数も件名で使えるので、`Re: {{topic}} について` のようなパターンも設定できます。


[](#%E6%8C%BF%E5%85%A5%E4%BD%8D%E7%BD%AE%E3%81%AE%E5%88%B6%E5%BE%A1) 挿入位置の制御

テンプレートの挿入位置は「カーソル位置」「本文先頭」「本文末尾（署名の手前）」から選択できます。返信メールの場合、署名の直前に挿入する設定にしておくと、引用部分を崩さずにテンプレートを差し込めます。


[](#%E3%81%9D%E3%81%AE%E4%BB%96%E3%81%AE%E6%A9%9F%E8%83%BD) その他の機能


**リッチテキスト対応**（Pro）: 太字・リスト・リンクなどの書式を含むテンプレートを作成可能

**インポート/エクスポート**（Pro）: JSON形式でバックアップ・復元ができる

**共有リンク**（Pro）: テンプレートをURLで他の人と共有可能

**利用統計**（Pro）: どのテンプレートをどれだけ使ったか、推定節約時間などをグラフで確認

**多言語対応**: 英語・日本語・ドイツ語・スペイン語・フランス語・イタリア語・韓国語・ポルトガル語に対応

**ダークモード対応**: GmailのテーマがダークならUIも自動で追従


無料プランでは10個までテンプレートを作成でき、基本的なテンプレート挿入・ショートカット・フォルダ整理が利用できます。


[](#%E6%8A%80%E8%A1%93%E7%9A%84%E3%81%AA%E8%A9%B1%EF%BC%9Agmail%E3%81%AEdom%E3%81%A8%E3%81%AE%E6%A0%BC%E9%97%98) 技術的な話：GmailのDOMとの格闘

Gmail拡張機能を開発したことがある方ならわかると思いますが、GmailのDOMは一筋縄ではいきません。ここからは、開発中に直面した技術的な課題をいくつか紹介します。


[](#%E3%82%BB%E3%83%AC%E3%82%AF%E3%82%BF%E3%81%AE%E5%A4%9A%E5%B1%A4%E3%83%95%E3%82%A9%E3%83%BC%E3%83%AB%E3%83%90%E3%83%83%E3%82%AF) セレクタの多層フォールバック

GmailのUIはReactなどのフレームワークで構築されておらず、独自のクロージャベースのレンダリングシステムで動いています。クラス名は `.AD` や `.nH` のような難読化された短い文字列で、Googleの内部ビルドシステムが生成しているため、アップデートのたびに変わる可能性があります。


SnapReplyでは、セレクタをプライマリとフォールバックの階層構造で管理しています。


```
const SELECTORS = {
  composeBody: {
    primary: '[role="textbox"][aria-label][g_editable="true"]',
    fallbacks: [
      'div[aria-label][contenteditable="true"][g_editable="true"]',
      '.Am.aiL',
      'div[contenteditable="true"].editable',
    ],
  },
  // ...
};
```

WAI-ARIAの `role` 属性やGmail固有の `g_editable` 属性を優先し、難読化クラス名はフォールバックとして残す設計です。これにより、GmailのUIが更新されても動作し続ける可能性を高めています。


[](#mutationobserver%E3%81%AE%E7%BD%A0) MutationObserverの罠

Gmailで新しい作成画面が開かれたことを検知するために `MutationObserver` を使用しています。ここでの注意点は、`attributes: true` を**絶対に指定しないこと**です。


```
this.observer.observe(document.body, {
  childList: true,
  subtree: true,
  // NEVER add attributes: true — causes infinite loops on Gmail
});
```

Gmailは属性の変更を大量に発生させるため、`attributes: true` を指定するとオブザーバーのコールバックが無限に呼ばれ、ブラウザタブがフリーズします。実際のコードにもコメントで警告を残しているほどです。加えて、コールバックはデバウンス処理（50ms）を挟んでパフォーマンスへの影響を抑えています。


[](#gmail%E3%81%AE%E3%82%AD%E3%83%BC%E3%83%9C%E3%83%BC%E3%83%89%E3%82%B7%E3%83%A7%E3%83%BC%E3%83%88%E3%82%AB%E3%83%83%E3%83%88%E3%81%A8%E3%81%AE%E8%A1%9D%E7%AA%81) Gmailのキーボードショートカットとの衝突

Gmailにはグローバルなキーボードショートカットがあり、入力フィールドにフォーカスしていても一部のキーイベントを横取りします。SnapReplyの検索バーや変数入力フォームで文字を打つと、Gmailが反応してしまう問題がありました。


対処としては、Shadow DOM内のUI要素からのキーイベントに対して `stopPropagation()` を呼び出しています。


```
const stopGmailCapture = (e: React.KeyboardEvent | React.CompositionEvent) => {
  e.stopPropagation();
};
```

`keydown`、`keyup`、`keypress` だけでなく、日本語入力（IME）に関わる `compositionstart`、`compositionupdate`、`compositionend` イベントもすべて伝播を止める必要があった点は、日本語圏ならではのハマりポイントでした。


[](#shadow-dom%E3%81%AB%E3%82%88%E3%82%8B%E3%82%B9%E3%82%BF%E3%82%A4%E3%83%AB%E9%9A%94%E9%9B%A2) Shadow DOMによるスタイル隔離

Content ScriptのUIはShadow DOM内にレンダリングしています。これにより、GmailのCSSとSnapReplyのCSSが互いに干渉しません。Tailwind CSSのスタイルは `CSSStyleSheet` を使ってShadow Root内に注入しています。


```
const sheet = new CSSStyleSheet();
sheet.replaceSync(`:host { all: initial; }\n${contentCss}`);
shadowRoot.adoptedStyleSheets = [sheet];
```

`:host { all: initial; }` で親ページのスタイルを完全にリセットしている点がポイントです。


[](#%E3%83%86%E3%83%B3%E3%83%97%E3%83%AC%E3%83%BC%E3%83%88%E6%8C%BF%E5%85%A5%E3%81%A8-execcommand) テンプレート挿入と `execCommand`


Gmailの作成画面は `contenteditable` なリッチテキストエディタです。テキストを挿入する際、単純に `innerHTML` を書き換えるとUndo履歴が壊れます。そこで、まず `document.execCommand('insertHTML')` を試し、失敗した場合にRange APIにフォールバックする二段構えの実装にしています。


`execCommand` は非推奨APIですが、`contenteditable` 環境でUndo対応のHTML挿入を行うには現時点で最も確実な手段です。


[](#%E3%81%BE%E3%81%A8%E3%82%81) まとめ

SnapReplyは「Gmailでの定型返信を、できるだけ少ない操作で終わらせる」ことだけに集中した拡張機能です。外部サービスへの接続は不要で、テンプレートデータはすべてブラウザのローカルストレージに保存されます。


無料で10テンプレートまで使えるので、日々のメール返信を効率化したい方はぜひ試してみてください。
