インボイス制度が始まって以来、請求書まわりの業務負担が増えた方は多いと思います。


- 毎月届くPDF請求書を1枚ずつ開いて、金額・日付・取引先名を会計ソフトに手入力

- さらに「T+13桁」の登録番号を目視確認して、適格請求書かどうかを調べる

- 月100枚以上処理する企業では、担当者が月平均**10時間以上**を手入力に費やしている（freee調査）


会計ソフトの請求書OCRサービスを使えば解決できますが、invox受取請求書なら月980円＋従量課金、マネーフォワードやfreeeは月2,000〜3,000円〜と、コストが気になるところです。


そこで、**完全無料で使えるChrome拡張機能**として「請求書よみとり」を作りました。


[https://chromewebstore.google.com/detail/請求書よみとり-pdf請求書をai-ocrで自動デ/kdlmfhdnnpbiogcbajgjheobihhanpae?authuser=0&hl=ja](https://chromewebstore.google.com/detail/%E8%AB%8B%E6%B1%82%E6%9B%B8%E3%82%88%E3%81%BF%E3%81%A8%E3%82%8A-pdf%E8%AB%8B%E6%B1%82%E6%9B%B8%E3%82%92ai-ocr%E3%81%A7%E8%87%AA%E5%8B%95%E3%83%87/kdlmfhdnnpbiogcbajgjheobihhanpae?authuser=0&hl=ja)


[](#%E8%AB%8B%E6%B1%82%E6%9B%B8%E3%82%88%E3%81%BF%E3%81%A8%E3%82%8A-%E3%81%A8%E3%81%AF) 請求書よみとり とは

**ChromeでPDFを開くだけで、AI-OCRが請求書データを自動抽出するChrome拡張機能**です。


抽出できる項目：


- 📅 発行日・支払期日

- 🏢 取引先名（発行者・宛先）

- 💰 合計金額・小計・消費税額

- 📊 税率区分（10%・8%軽減税率）

- 🔢 適格請求書発行事業者登録番号（T+13桁）

- 📝 明細行（品目・数量・単価）


抽出したデータはCSV/Excel形式でエクスポートできるので、freee・マネーフォワード・弥生などの会計ソフトへそのまま取り込めます。


[](#%E3%81%93%E3%82%93%E3%81%AA%E6%96%B9%E3%81%AB%E5%88%BA%E3%81%95%E3%82%8A%E3%81%BE%E3%81%99) こんな方に刺さります


**フリーランス・個人事業主** — 確定申告前に1年分の請求書を整理したい

**経理担当者** — 月末の請求書処理を少しでも時短したい

**中小企業オーナー** — 会計ソフトの月額費用をかけずにOCRを使いたい

**税理士・会計士** — 顧問先から送られてくるPDF請求書をまとめてデータ化したい

**Chrome拡張ならではの強みは3つです。**


**ゼロコストで始められる** — 月3件まで永久無料。アカウント登録も不要

**インストール即使える** — Chromeに追加して、PDFを開くだけ

**会計ソフト非依存** — freeeでもマネーフォワードでも弥生でもExcelでも、CSVで取り込める


[](#%E4%B8%BB%E3%81%AA%E6%A9%9F%E8%83%BD%E8%A9%B3%E7%B4%B0) 主な機能詳細

[](#%F0%9F%A4%96-ai-ocr%E8%87%AA%E5%8B%95%E8%AA%AD%E3%81%BF%E5%8F%96%E3%82%8A) 🤖 AI-OCR自動読み取り

pdf.js → GPT-4o-mini Visionの3層アーキテクチャを採用。


**第1層**: pdf.jsでテキスト埋め込みPDFからテキスト抽出

**第2層**: テキストが取れない場合（スキャンPDF等）、画像化してOCR処理

**第3層**: GPT-4o-miniで構造化データに変換


手書き風フォントや複雑なレイアウトの請求書にも対応。処理はすべてクライアントサイドで完結するため、**サーバーにデータが残りません**。


[](#%E2%9C%85-%E3%82%A4%E3%83%B3%E3%83%9C%E3%82%A4%E3%82%B9%E5%88%B6%E5%BA%A6%E3%81%B8%E3%81%AE%E5%AF%BE%E5%BF%9C) ✅ インボイス制度への対応

2023年10月開始のインボイス制度（適格請求書等保存方式）に対応。登録番号（T+13桁）を自動読み取りします。


経過措置についても確認できます：


- 2023年10月〜2026年9月：非登録事業者からの仕入れは80%控除

- 2026年10月以降：70%控除に変更予定（2026年度税制改正大綱）


!

**2026年10月に経過措置が変わります。**

現在80%控除の経過措置が、2026年10月から70%に引き下げられる予定です。取引先が適格請求書発行事業者かどうかの確認が、これまで以上に重要になります。

[](#%E6%8A%80%E8%A1%93%E3%82%B9%E3%82%BF%E3%83%83%E3%82%AF) 技術スタック
開発者向け詳細


**フロントエンド**: React 18 + TypeScript + Vite + Tailwind CSS

**Chrome拡張**: Manifest V3 + CRXJS

**PDF処理**: pdf.js（テキスト抽出）

**OCR**: OpenAI GPT-4o-mini Vision API（BYOK方式）

**決済**: ExtensionPay（Stripe連携）


処理はすべてクライアントサイドで完結。サーバーコストゼロで運用できる構成です。


[](#%E3%82%88%E3%81%8F%E3%81%82%E3%82%8B%E8%B3%AA%E5%95%8F) よくある質問

[](#q.-%E3%82%B9%E3%82%AD%E3%83%A3%E3%83%B3%E3%81%97%E3%81%9Fpdf%EF%BC%88%E7%94%BB%E5%83%8Fpdf%EF%BC%89%E3%81%AB%E3%82%82%E5%AF%BE%E5%BF%9C%E3%81%97%E3%81%A6%E3%81%84%E3%81%BE%E3%81%99%E3%81%8B%EF%BC%9F) Q. スキャンしたPDF（画像PDF）にも対応していますか？

はい。テキストが埋め込まれていないスキャンPDFでも、AI-OCRで画像からテキストを認識します。ただし、手書きや極端に画質が悪いPDFは精度が落ちる場合があります。


[](#q.-%E7%99%BB%E9%8C%B2%E7%95%AA%E5%8F%B7%E3%81%AE%E7%85%A7%E5%90%88%E3%81%AF%E3%81%A9%E3%81%86%E3%82%84%E3%81%A3%E3%81%A6%E3%81%84%E3%82%8B%E3%81%AE%EF%BC%9F) Q. 登録番号の照合はどうやっているの？

国税庁が公開している「適格請求書発行事業者公表システムWeb-API」を利用して、リアルタイムで登録番号の有効性を確認しています。


[](#q.-%E3%83%87%E3%83%BC%E3%82%BF%E3%81%AF%E3%81%A9%E3%81%93%E3%81%AB%E4%BF%9D%E5%AD%98%E3%81%95%E3%82%8C%E3%81%BE%E3%81%99%E3%81%8B%EF%BC%9F) Q. データはどこに保存されますか？

すべてのデータはブラウザ内（`chrome.storage.local`）に保存されます。外部サーバーには送信・保存されません。拡張機能をアンインストールすると、すべてのデータが削除されます。


[](#q.-openai-api%E3%82%AD%E3%83%BC%E3%81%8C%E3%81%AA%E3%81%84%E3%81%A8%E4%BD%BF%E3%81%88%E3%81%BE%E3%81%9B%E3%82%93%E3%81%8B%EF%BC%9F) Q. OpenAI APIキーがないと使えませんか？

基本的なテキストPDF（テキストが埋め込まれているPDF）はAPIキーなしでも読み取れます。スキャンPDFや複雑なレイアウトの処理にはOpenAI APIキーが必要です。


[](#%E3%81%8A%E3%82%8F%E3%82%8A%E3%81%AB) おわりに

invoxやfreeeのような高機能なSaaSは素晴らしいですが、「月数枚しか処理しない」「まず試してみたい」という方には、コストゼロで使えるChrome拡張のほうが合っているケースもあります。


2026年10月にインボイスの経過措置が変わるタイミングで、取引先の登録番号チェックをきちんと整備しておくのもよいかもしれません。
