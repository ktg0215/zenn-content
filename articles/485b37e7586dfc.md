[](#%E3%81%AF%E3%81%98%E3%82%81%E3%81%AB) はじめに

Chrome拡張を17本公開してきた個人開発者のしょうたです。


先日、**Dev Tools Hub**（[https://tools.dev-tools-hub.xyz](https://tools.dev-tools-hub.xyz)）という開発者向けの無料オンラインツール集を公開しました。JSONフォーマッター、Base64エンコーダー、正規表現テスターなど10個のツールを1つのサイトにまとめたものです。


この記事では「なぜ作ったか」から「技術スタックの選定」「デプロイ」「マーケティング」まで、やったこと全部を包み隠さず書きます。


[https://tools.dev-tools-hub.xyz](https://tools.dev-tools-hub.xyz)


[](#%E3%81%AA%E3%81%9C%E4%BD%9C%E3%81%A3%E3%81%9F%E3%81%8B) なぜ作ったか

開発中に毎回こういうことをやっていました。


- JSONを整形したい → jsonformatter.org を開く

- Base64をデコードしたい → base64decode.org を開く

- 正規表現をテストしたい → regex101.com を開く

- タイムスタンプを変換したい → epochconverter.com を開く


**ブラウザのタブが爆発する。**


それぞれのサイトは優秀なのですが、毎回サイトを探して開くのが面倒でした。しかも広告がうるさかったり、UIが古かったり。


「全部1つのサイトで済めばいいのに」と思っていたので、作りました。


[](#%E6%8A%80%E8%A1%93%E3%82%B9%E3%82%BF%E3%83%83%E3%82%AF) 技術スタック

[](#%E3%83%95%E3%83%AC%E3%83%BC%E3%83%A0%E3%83%AF%E3%83%BC%E3%82%AF%EF%BC%9Anext.js-15%EF%BC%88app-router-%2B-ssg%EF%BC%89) フレームワーク：Next.js 15（App Router + SSG）

Vite + Reactも検討しましたが、**SEOのために静的HTMLを事前生成したかった**のでNext.jsを選択。各ツールページが「JSON整形 オンライン」「Base64デコード」などのキーワードで検索上位に入ることを狙っています。


```
フレームワーク：Next.js 15（App Router、SSG）
言語：TypeScript
スタイリング：Tailwind CSS v4
多言語：next-intl（日本語・英語対応）
パッケージ管理：pnpm
```

[](#%E3%83%9B%E3%82%B9%E3%83%86%E3%82%A3%E3%83%B3%E3%82%B0%EF%BC%9Acloudflare-pages) ホスティング：Cloudflare Pages

当初Vercelを検討していましたが、**Vercelの無料プランは商用利用不可**であることが判明。広告を貼る予定のサイトなので有料プラン（$20/月）が必要になります。


代わりに**Cloudflare Pages**を選択しました。商用利用OK、帯域無制限、無料。Next.jsのSSG（静的エクスポート）ならCloudflare Pagesで問題なく動きます。


```
next.config.ts に output: 'export' を設定
→ npm run build で out/ ディレクトリに静的ファイルを出力
→ GitHub にpush → Cloudflare Pages が自動デプロイ
```

GitHubのmainブランチにpushするだけで自動デプロイされる体制を最初に作ったので、開発が快適でした。


[](#%E3%83%89%E3%83%A1%E3%82%A4%E3%83%B3%EF%BC%9Adev-tools-hub.xyz) ドメイン：dev-tools-hub.xyz

既存のS-HubブランドサイトがXserverで動いていたため、ネームサーバーをXserverからCloudflareに移管しました。`tools.dev-tools-hub.xyz` をサブドメインとしてCloudflare Pagesに割り当てる形です。


[](#%E3%82%A2%E3%83%8A%E3%83%AA%E3%83%86%E3%82%A3%E3%82%AF%E3%82%B9%EF%BC%9Aga4-%2B-cloudflare-web-analytics) アナリティクス：GA4 + Cloudflare Web Analytics

開発者ユーザーは広告ブロッカー使用率が70%以上と言われています。Google Analyticsだけだと実際のトラフィックの30〜40%しか計測できない可能性があります。


そのため**Cloudflare Web Analytics**を併用しています。Cloudflareのインフラレベルで計測するため、広告ブロッカーに引っかかりにくいです。しかも無料。

**合計：約7〜8日で10ツール完成。**


外部ライブラリをほぼ使わない設計にしたので、バンドルサイズが小さく、ページの読み込みが速いです。


[](#%E5%AE%9F%E8%A3%85%E3%81%A7%E5%B7%A5%E5%A4%AB%E3%81%97%E3%81%9F%E7%82%B9) 実装で工夫した点

**入力値のLocalStorage保持**


各ツールのテキストエリアに入力した値を自動でLocalStorageに保存しています。ページを閉じて再度開いても前回の入力値が残っているので、作業の続きをすぐ再開できます。


**URLパラメータでの共有**


入力値をURLパラメータにエンコードして共有できます。たとえばJSONフォーマッターで整形したJSONをURLとして同僚に共有できます。


**ダークモードデフォルト**


開発者向けサイトなのでダークモードをデフォルトにしました。`prefers-color-scheme`に従って初期テーマを決定し、手動切り替えも可能です。


[](#%E5%A4%9A%E8%A8%80%E8%AA%9E%E5%AF%BE%E5%BF%9C%EF%BC%88%E6%97%A5%E6%9C%AC%E8%AA%9E%E3%83%BB%E8%8B%B1%E8%AA%9E%EF%BC%89) 多言語対応（日本語・英語）

`next-intl`を使ってルーティングレベルで多言語対応しています。


```
/ja/json-formatter → 日本語版
/en/json-formatter → 英語版
```

URLスラッグは全言語で英語統一。日本語スラッグにするとエンコード問題やリンク共有の不便さが発生するためです。


翻訳対象はUIラベル・エラーメッセージ・ページのSEOメタデータのみ。JSONやBase64などの技術用語はどの言語でも同じなので翻訳不要です。


[](#seo%E6%88%A6%E7%95%A5) SEO戦略

ツール系サイトのSEOは「各ツールページが独立したSEO対象になる」という特性があります。


`/ja/json-formatter` → 「JSON整形」で検索上位を狙う

`/ja/base64` → 「Base64エンコード」で検索上位を狙う

`/ja/regex-tester` → 「正規表現テスト」で検索上位を狙う


ツールを増やすほど検索流入の間口が広がるので、まずはツール数を増やすことに集中しています。


各ツールページには以下を設置しています：


- 適切なtitle・meta descriptionの設定

`WebApplication`スキーマの構造化データ（JSON-LD）

- FAQPageスキーマ

- hreflangの設定（日英の相互参照）

[](#hacker-news%E3%80%8Cshow-hn%E3%80%8D%E3%81%AB%E6%8A%95%E7%A8%BF) Hacker News「Show HN」に投稿

公開と同時に**Hacker News**の「Show HN」に投稿しました。


HNはエンジニア向けの情報共有サイトで、「Show HN」は自分で作ったプロダクトを紹介するためのカテゴリです。フロントページに上がれば数千〜数万のトラフィックが1日で来ます。


投稿のコツとして学んだこと：


- タイトルはマーケティング用語を使わない

- 投稿直後にファーストコメントで「なぜ作ったか・技術的な判断・正直な制限」を語る

- コメントには即座に返信する


!

HNへの投稿はタイミングも重要で、**火〜木曜の22:00〜24:00（日本時間）**が米国東海岸の朝9時〜10時に相当し、最も反応が良いとされています。


[](#github-awesome-lists%E3%81%B8%E3%81%AEpr) GitHub Awesome-listsへのPR

`burningtree/awesome-json`（スター1,700+）にPRを送りました。マージされれば永続的なバックリンクとトラフィックが入ります。


PR送信はClaude Codeに `gh` CLIを使って実行してもらいました。PRの作成自体は5分もかからず完了。

[](#%E5%8F%8E%E7%9B%8A%E3%83%A2%E3%83%87%E3%83%AB) 収益モデル

**Google AdSense**をメインの収益源として考えています。完全無料・登録不要のツールサイトなので、有料プランは設けません。


ただし開発者ユーザーは広告ブロッカー使用率が高いため、実際の収益はPV数の30〜40%分程度になると予測しています。月間10万PVで月額1〜3万円程度が現実的な目安です。


将来的にはCarbon Ads（開発者向け広告ネットワーク）やスポンサーシップへの切り替えも検討しています。
